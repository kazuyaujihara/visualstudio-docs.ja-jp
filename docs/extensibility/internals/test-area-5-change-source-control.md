# <a name="test-area-5-change-source-control"></a>テスト領域 5: ソース管理の変更
このソース管理プラグインのテスト領域では、 **[ソース管理の変更]** コマンドを使用したソース管理の変更について説明します。

 **[ソース管理の変更]** コマンドでは、ユーザーに4つの基本機能が用意されています。

- **束縛**

   ソリューション/プロジェクトとバージョンストア間のソース管理リンクを確立または再確立することをユーザーに許可します。

- **Unbind**

   接続ごとにソース管理からプロジェクトまたはソリューションを削除します。

- **接続/切断:**

  管理されたソリューションの接続状態またはオフライン状態を切り替えます。これについては、「区分3」で説明しています。 詳細については、「[テスト領域 3: チェックアウト/](../../extensibility/internals/test-area-3-check-out-undo-checkout.md)チェックアウトの取り消し」を参照してください。

## <a name="command-menu-access"></a>コマンドメニューへのアクセス
 テストケースでは、次の [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 統合開発環境メニューパスが使用されます。

 ソース管理の変更:**ファイル**、**ソース管理**、**ソース管理の変更**。

### <a name="case-5b-unbind"></a>ケース 5b: バインド解除
 バインドを解除すると、プロジェクトとそのソリューションからソースコード管理情報が削除されます。 影響を受けるプロジェクトとソリューションは、ユーザー選択の組み合わせと、項目がソース管理に追加された方法に基づいています。

|操作|テストステップ|確認が必要な結果|
|------------|----------------|--------------------------------|
|1つのファイルシステムまたはローカル IIS Web プロジェクトと1つのクライアントプロジェクトを含むソリューションのバインドを解除する|1. ファイルシステムまたはローカル IIS Web プロジェクトを作成します。<br />2. ソース管理にソリューションを追加します。<br />3. ソリューションに新しいクライアントプロジェクトを追加します。<br />4. メッセージが表示されたら、ソリューションのチェックアウトを受け入れます。<br />5. **[ソース管理の変更]** ダイアログボックスを開きます。<br />6. **[バインド解除]** をクリックします。<br />7. **[OK]** をクリックしてダイアログボックスを閉じます。<br />8. ソリューション、プロジェクト、ソリューション項目、プロジェクト項目をチェックアウトしようとしています。|ソリューションとプロジェクトがソース管理下にありません。<br /><br /> [ソース管理] メニューのコマンドは表示されません。|
