# <a name="c6320"></a>C6320
警告 C6320: 例外フィルター式は定数 EXCEPTION_EXECUTE_HANDLER です。 これは、処理されることを意図していない例外をマスクする可能性があります。

 この警告は、\_\_except ブロックで EXCEPTION_EXECUTE_HANDLER 定数を使用しての副作用を示します。 この場合は、ステートメントで、 \_\_except ブロックが常に特定の関数で処理しない例外など、例外を処理するために実行します。 これを処理する前に例外を確認することをお勧めします。

## <a name="example"></a>例
 次のコードは、\_\_except ブロックしているために、この警告のすべての種類の例外を処理しようとしています。

```
#include <stdio.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;  // might cause access violation exception
      // code ...
   }
   __except(EXCEPTION_EXECUTE_HANDLER) // warning
   {
      puts("in except");
      // code ...
   }
}
```

 この警告を解決するには使用`GetExceptionCode`に次のコードに示すように処理する前に特定の例外を確認します。

```
#include <stdio.h>
#include <windows.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;    // might cause access violation exception
      // code ...
   }
   __except(GetExceptionCode()==EXCEPTION_ACCESS_VIOLATION ?
               EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH)
   {
      puts("in except");
      // code ...
   }
}
```
