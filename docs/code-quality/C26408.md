# <a name="c26408-no_malloc_free"></a>C26408 NO_MALLOC_FREE
この警告フラグは、R に従って `malloc` または `free` が明示的に呼び出される場所を示します。@No__t-0 と `free` は避けてください。 このような警告の問題の1つとして、 [std:: make_unique](/cpp/standard-library/memory-functions#make_unique)を使用して、オブジェクトの明示的な作成と破棄を回避することが考えられます。 このような修正が許容されない場合は、operator [new と delete](/cpp/cpp/new-and-delete-operators)を使用することをお勧めします。 場合によっては、例外が開始されない場合、`malloc` および `free` は、nothrow バージョンの @no__t 演算子 (-2 および `delete`) に置き換えることができます。

## <a name="remarks"></a>コメント
- Malloc () を検出するには、呼び出しによって "malloc" または "std:: malloc" という名前のグローバル関数が呼び出されるかどうかを確認します。 関数は `void` へのポインターを返し、符号なし整数型の1つのパラメーターを受け入れなければなりません。
- Free () を検出するには、"free" または "std:: free" という名前のグローバル関数を確認します。この場合、結果は返されず、1つのパラメーターを受け取ります。これは `void` へのポインターです。
