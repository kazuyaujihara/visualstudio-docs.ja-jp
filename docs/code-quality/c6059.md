# <a name="c6059"></a>C6059
警告 C6059: \<function > への呼び出しでの長さのパラメーターが正しくありません。 @No__t_0variable のバッファーサイズではなく、残りの文字数を渡し >

 この警告は、文字列連結関数の呼び出しが、連結する文字数に対して正しくない値を渡す可能性があることを示しています。 この欠陥により、利用可能なバッファーオーバーランまたはクラッシュが発生する可能性があります。 この障害の一般的な原因として、バッファー内の残りの文字数ではなく、バッファーサイズが文字列操作関数に渡されることが挙げられます。

## <a name="example"></a>例
 この警告が発生するコード例を次に示します。

```
#include <string.h>
#define MAX 25

void f( )
{
  char szTarget[MAX];
  char *szState ="Washington";
  char *szCity="Redmond, ";

  strncpy(szTarget,szCity, MAX);
  szTarget[MAX -1] = '\0';
  strncat(szTarget, szState, MAX); //wrong size
  // code ...
}
```

 この警告を修正するには、次のコードに示すように、正しい文字数を使用して連結します。

```
#include <string.h>
#define MAX 25

void f( )
{
  char szTarget[MAX];
  char *szState ="Washington";
  char *szCity="Redmond, ";

  strncpy(szTarget,szCity, MAX);
  szTarget[MAX -1] = '\0';
  strncat(szTarget, szState, MAX - strlen(szTarget)); // correct size
  // code ...
}
```

 安全な文字列操作関数を使用してこの警告を修正するには、次のコードを参照してください。

```
#include <string.h>

void f( )
{
  char *szState ="Washington";
  char *szCity="Redmond, ";

  size_t nTargetSize = strlen(szState) + strlen(szCity) + 1;
  char *szTarget= new char[nTargetSize];

  strncpy_s(szTarget, nTargetSize, szCity,strlen(szCity));
  strncat_s(szTarget, nTargetSize, szState,
                    nTargetSize - strlen(szTarget));
  // code ...
  delete [] szTarget;
}
```
